# 安全策略

| 版本 | 支持状态 |
| ---- | -------- |
| 最新版本 | ✅ 支持 |
| 上一个主要版本 | ✅ 支持 |

## 报告漏洞

我们非常重视安全问题。如果您发现了安全漏洞，请按照以下步骤报告：

### 1. 不要公开披露

**重要：** 请不要在 GitHub Issues 中公开报告安全漏洞。这可能会让其他用户面临风险。

### 2. 私密报告

请通过以下方式私密报告安全漏洞：

- **电子邮件：** [在此处添加您的安全邮箱地址]
- **GitHub Security Advisories：** 使用 GitHub 的安全公告功能

### 3. 报告内容

请在报告中包含以下信息：

- **漏洞描述：** 详细描述漏洞的性质和影响
- **重现步骤：** 提供重现漏洞的详细步骤
- **影响范围：** 说明漏洞可能影响的用户和系统
- **建议修复：** 如果您有修复建议，请一并提供
- **联系方式：** 您的联系方式，以便我们跟进

### 4. 响应时间

- **确认收到：** 我们会在 48 小时内确认收到您的报告
- **初步评估：** 我们会在 7 天内完成初步评估
- **修复时间：** 根据漏洞严重程度，我们会在 30-90 天内发布修复

## 安全最佳实践

### 对于用户

1. **保持更新：** 始终使用最新版本的 MediaUnlockTest
2. **验证下载：** 从官方渠道下载，并验证文件完整性
3. **网络安全：** 在使用代理或 VPN 时注意网络安全
4. **权限管理：** 不要以管理员权限运行，除非必要

### 对于开发者

1. **依赖管理：** 定期更新依赖包，及时修复已知漏洞
2. **代码审查：** 所有代码变更都需要经过安全审查
3. **输入验证：** 对所有用户输入进行严格验证
4. **错误处理：** 避免在错误信息中泄露敏感信息

## 安全更新

### 更新通知

- 安全更新会通过 GitHub Releases 发布
- 重要安全更新会在项目 README 中特别标注
- 建议用户订阅项目的 Release 通知

### 更新流程

1. **漏洞确认：** 确认漏洞的存在和影响
2. **修复开发：** 开发安全修复补丁
3. **测试验证：** 充分测试修复的有效性
4. **发布更新：** 发布新版本并通知用户

## 安全团队

我们的安全团队由以下成员组成：

- **项目维护者：** 负责整体安全策略和决策
- **代码审查者：** 负责代码安全审查
- **安全响应者：** 负责处理安全报告和协调修复
